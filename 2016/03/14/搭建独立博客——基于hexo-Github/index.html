搭建独立博客——基于Hexo+Github

  • Github
  • Hexo

在自学Android应用开发的时候，受启发于某位大大，自己也开始学习搭建个人blog，以此来记录自学过程中的经验以及教训吧。

但是为什么不用现成的可视化编辑博客平台呢？因为很多博客平台上面点进去广告太多，阅读体验太差。作为一个有编程经验的人，自己搭建一个博客不是一件难事。

回到正题，因为Github上面有很多已经设计好的博客模板，还可以免为托管，所以使用起来非常方便。下面来说说如何搭建自己的个人Blog。

一. Windows环境下Hexo的搭建

1. 安装Git

到Git官网下载即可。

博主使用的当前版本是v2.7.2

【PS: 安装进行到”Select Components”这一步时记得确认将“Git Bash here”和“Git GUI here”这两个选项打勾。以方便后面的操作。】

2. 安装Node.js

同样，到Node.js官网下载最新版本。

博主使用的当前版本是v5.8.0

【PS: 最好安装在默认路径。】

3. 安装Hexo

前面的Git和Node安装好后，在屏幕的任意一处点击右键，在菜单中选择”Git Bash”。

【PS:在v2.7.2版本的Git中复制操作是”Ctrl+ins”,粘贴操作是”Shift+ins”，而”Ctrl+C”会终止当前操作】

输入以下命令：

npm install -g hexo

应该会有人在执行这一步之后程序就会出现ERR ERR ERR…博主当时也卡在这里，找了无数种方法，最后才发现原来是下载hexo的网址被墙了，程序无法下载。这时候就要给npm下载地址换源了。下面介绍两种方法：

  1. 分步输入以下两个命令：

    npm install -g cnpm –registry=https://registry.npm.taobao.org
    cnpm install hexo-cli -g`

  2. 编辑”C:\Users\你的用户名”目录下的”.npmrc”文件
    加入如下内容

    registry = http://registry.cnpmjs.org

    再在Git Bash中输入命令npm install -g hexo即可

PS: 推荐使用第二种方法，因为后面的操作中也会出现下载时被墙的问题。

4. 创建Hexo目录

可在任意地方新建一个文件夹，此文件夹就是你的个人Blog包括主要信息、文章、主题等文件所存放的地方，建议用全英路径一面出现不必要的ERR。例如：”C:\HEXO”。
建好文件夹后，在文件夹内点击右键，然后点击”Git Bash Here”。

  • 输入命令

    hexo init

    完成文件夹内会自动生成搭建一个hexo博客所需要的文件。如果执行完后命令框显示如下，那就证明已经成功生成一个hexo目录。

  • 输入命令

    npm install

    npm会自动安装依赖环境。

    【PS: 若上一步命令”hexo init”执行成功，可跳过此步】

5. 本地运行个人Blog

以上操作全部成功后，即可在本地试运行个人Blog。

  • 输入命令

    hexo s


在浏览器中输入网址” http://localhost:4000/“ 即可浏览到刚刚生成的blog。


至此，Windows环境下Hexo的搭建就已经全部完成了。

二. 基于GitHub Page的Hexo搭建

这一部分博主推荐一篇文章，上面已经详细的说明了如何设置GitHub以及如何将本地Hexo目录与GitHub关联起来。

Hexo系列教程: (三)基于GitHub Page的Hexo搭建

需要注意的是：

  1. 这里推荐下载一个Notepad++来打开后缀为”.yml”以及”.md”等的文件而不要使用记事本打开，以避免出现乱码，尤其是在编辑”_config.yml”时，用记事本编辑输入中文，在Blog中会显示成乱码。

    解决方法就是用Noteapp以UTF-8格式编码编辑文件。

  2. 若在安装Hexo时没有选择编辑”.npmrc”的方法，那么在最后一步执行hexo deloy时可能出会出现npm ERR!。

    这个问题前面也提到过，在大天朝，下载源的地址被墙了。

    解决方法与之前相同，编辑”C:\Users\你的用户名”目录下的”.npmrc”文件
    加入如下内容

    registry = http://registry.cnpmjs.org

    然后在Hexo目录下执行

    hexo g
    hexo s

    至此，你的个人Blog就已经完全搭建完毕了。

三. 发表你的第一篇文章

同样的，在Hexo目录下输入以下命令

hexo new “文章名”

然后在”source/_posts”目录下面，多了一个”文章名.md”的文件。
打开之后会看到：


文件的开头是属性，采用统一的yaml格式，用三条短横线分隔。下面是文章正文。

文章的正文支持markdown格式，建议你先学习一下它的语法。markdown不像html有一大堆标签，很简单，只有几个符号。

新建、删除或修改文章后，不需要重启hexo server，刷新一下即可预览。

如果已经编辑好文章，就执行以下命令将文章同步到自己的Blog

hexo clean && hexo g && hexo deploy && hexo clean

然后登陆自己的Blog主页，就可以看到自己发表的第一篇博文了。

四. 总结

其实搭建一个个人Blog会遇到很多很多的坑，因为是在大天朝啊，各种各样的下载源地址都被墙了，在Windows上搭建Hexo还有最后一步将Hexo上传到GitHub的时候博主也是找了无数种解决方法，尝试了无数次才成功了，当时第一次深深地体会到了在大天朝被墙的痛苦。

不过还在最后终于完成了，写这篇博文是为了记录一下，也为了让看到这篇博文的人少踩些坑吧。

其实博主也是刚刚入坑，很多Hexo的功能还在摸索当中。
