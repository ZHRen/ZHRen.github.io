搭建独立博客——基于Hexo+Github

  • Github
  • Hexo

在自学Android应用开发的时候，受启发于某位大大，自己也开始学习搭建个人blog，以此来记录自学过程中的经验以及教训吧。
但是为什么不用现成的可视化编辑博客平台呢？因为很多博客平台上面点进去广告太多，阅读体验太差。作为一个有编程经验的人，自己搭建一个博客不是一件难事。
回到正题，因为Github上面有很多已经设计好的博客模板，还可以免为托管，所以使用起来非常方便。下面来说说如何搭建自己的个人Blog。

Windows环境下Hexo的搭建

1. 安装Git

到Git官网下载即可。
博主使用的当前版本是v2.7.2
【PS: 安装进行到”Select Components”这一步时记得确认将“Git Bash here”和“Git GUI here”这两个选项打勾。以方便后面的操作。】

2. 安装Node.js

同样，到Node.js官网下载最新版本。
博主使用的当前版本是v5.8.0
【PS: 最好安装在默认路径。】

3. 安装Hexo

前面的Git和Node安装好后，在屏幕的任意一处点击右键，在菜单中选择”Git Bash”。
【PS:在v2.7.2版本的Git中复制操作是”Ctrl+ins”,粘贴操作是”Shift+ins”，而”Ctrl+C”会终止当前操作】

输入以下命令：
npm install -g hexo

应该会有人在执行这一步之后程序就会出现ERR ERR ERR…博主当时也卡在这里，找了无数种方法，最后才发现原来是下载hexo的网址被墙了，程序无法下载。这时候就要给npm下载地址换源了。下面介绍两种方法：

  1. 分步输入以下两个命令：
    npm install -g cnpm --registry=https://registry.npm.taobao.org
    cnpm install hexo-cli -g
  2. 编辑”C:\Users\你的用户名”目录下的”.npmrc”文件
    加入如下内容
    registry = http://registry.cnpmjs.org
    再在Git Bash中输入命令npm install -g hexo即可

PS: 推荐使用第二种方法，因为后面的操作中也会出现下载时被墙的问题。

4. 创建Hexo目录

可在任意地方新建一个文件夹，此文件夹就是你的个人Blog包括主要信息、文章、主题等文件所存放的地方，建议用全英路径一面出现不必要的ERR。例如：”C:\HEXO”。
建好文件夹后，在文件夹内点击右键，然后点击”Git Bash Here”。

  • 输入命令 hexo init
    完成文件夹内会自动生成搭建一个hexo博客所需要的文件。如果执行完后命令框显示如下，那就证明已经成功生成一个hexo目录。
  • 输入命令 npm install
    npm会自动安装依赖环境。
    【PS: 若上一步命令”hexo init”执行成功，可跳过此步】

5. 本地运行个人Blog

以上操作全部成功后，即可在本地试运行个人Blog。

  • 输入命令 hexo s


在浏览器中输入网址” http://localhost:4000/“ 即可浏览到刚刚生成的blog。
